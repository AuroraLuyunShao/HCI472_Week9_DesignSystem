Button

Button directs the users to a new page or the prototype.It is placed at the project overview section of each case study.

Example

The example is the same size across all devices. Hover to examine its hover effect.

Accessibility

In most cases, <button> should be used for clickable areas fpr the greatest clarity.

  • Avoid using <div> or <img> tags to create buttons. Screen readers generally wonâ€™t know that either is a usable button.
  • If must use a different HTML element to do the same functionality as a button, use the aria role='button' to let screen readers know the area is clickable.
  • Consider using a input for a button(especially used in forms) as long as it has a type='button'.

Usage

Buttons should be used in a consistent style across the website.

  • Make sure there is a hover effect.
  • Make sure the user understand where the button links to.
  • Include a shadow to indicate the button is an active one.

Code

        
          <button type="button" class="btn btn-primary">Primary</button>
